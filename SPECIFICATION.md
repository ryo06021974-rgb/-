# チーム役割診断アプリ 仕様書 Ver1.1

## 1. アプリ概要

本アプリは、ユーザーが24問の質問に回答することで、チーム活動においてどのような役割を担いやすいかを診断するWebアプリである。

一般的な性格診断ではなく、以下を目的とする。

- 自分の強みとなる役割を知る
- チーム活動での適性を知る
- 体育祭・文化祭・グループ研究などにおける活躍の仕方を知る

診断結果では最も適性の高い役割を表示し、あわせて他の役割との割合や相性の良い役割も表示する。

## 2. 開発目的

大学の授業・ゼミ・ポートフォリオとして活用できるレベルのWebアプリを作成する。

また、将来的に以下へ拡張できる構造にする。

- クラス全体の診断結果集計
- CSV出力
- チーム編成分析

## 4. 診断対象役割

診断対象は以下の8役割とする。

| 役割 | 絵文字 | 概要 |
| --- | --- | --- |
| リーダー | 👑 | 全体をまとめ方向性を示す |
| プランナー | 📋 | 計画や段取りを考える |
| アイデアマン | 💡 | 新しい発想を生み出す |
| ムードメーカー | 🎉 | 場の雰囲気を良くする |
| サポーター | 🤝 | 周囲を支えフォローする |
| アクション | 🚀 | まず行動に移す |
| アナリスト | 🔍 | 分析し品質を高める |
| チャレンジャー | 🔥 | 挑戦や変化を好む |

## 5. 診断方式

### 質問数

24問

### 回答形式

4択

### 得点方式

各選択肢に対応する役割を割り当てる。

ユーザーが選択した選択肢に対応する役割へ1点加算する。

例:

```python
{
    "question": "修学旅行の自由行動時間。あなたはどう過ごしたい？",
    "choices": {
        "事前にルートや時間を細かく決めておく": "プランナー",
        "現地で面白そうな場所を見つけながら行動する": "アイデアマン",
        "みんなが楽しめるよう相談しながら決める": "サポーター",
        "とりあえず気になる場所へ向かう": "アクション"
    }
}
```

## 6. 公平性ルール

24問 x 4択 = 96選択肢

診断対象役割 = 8種類

96 / 8 = 12

よって各役割は必ず12回ずつ登場する。

| 役割 | 出現回数 |
| --- | --- |
| リーダー | 12 |
| プランナー | 12 |
| アイデアマン | 12 |
| ムードメーカー | 12 |
| サポーター | 12 |
| アクション | 12 |
| アナリスト | 12 |
| チャレンジャー | 12 |

## 7. 質問設計方針

役割が直接分からない質問にする。

### NG例

文化祭であなたは？

- 全体をまとめる
- 計画を立てる
- アイデアを出す
- 行動する

### OK例

文化祭前日、準備していた企画の一部が使えなくなった。

あなたが最初に考えるのは？

- 今後の進行への影響
- 別の方法で代用できないか
- 誰か困っている人はいないか
- とりあえず現場を見に行く

## 9. 診断結果表示

結果画面の表示順は以下とする。

### ① メイン役職表示

最も得点が高い役職を表示する。

例:

```text
👑 リーダータイプ
```

### ② 他役職との割合表示

例:

```text
👑 リーダー 28%
📋 プランナー 20%
🤝 サポーター 18%
🚀 アクション 12%
...
```

pandasを利用する。

```python
result_df = pd.DataFrame(...)
```

表示:

```python
st.dataframe(result_df)
```

### ③ グラフ表示

pandasのDataFrameを利用する。

```python
chart_df = pd.DataFrame(...)
```

表示:

```python
st.bar_chart(chart_df)
```

### ④ メイン役職の概要

例:

#### 👑 リーダー

チームを導く統率者

あなたは周囲の状況を把握しながら、自然と方向性を示すことができます。

文化祭や体育祭など、多くの人が関わる場面で活躍します。

強み:

- 統率力
- 責任感
- 決断力

### ⑤ 相性機能

固定表示とする。

例:

- 📋 プランナー
- 🤝 サポーター

## 12. Streamlit機能
